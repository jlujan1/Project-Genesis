Here is a step-by-step framework to build the "Project Genesis" simulation.

---

### Phase 1: The Virtual Substrate (The Sandbox)

Consciousness needs something to be conscious *of*. Before we build the brain, we must build the universe it lives in.

* **The Physics Engine:** We deploy a highly robust, realistic 3D virtual environment. It must have strict physical rules: gravity, light mapping, sound wave propagation, and collision physics.
* **The Avatar:** We create a digital "body" for our agent. It needs simulated sensors (virtual cameras for eyes, audio receivers for ears, and collision detectors for touch) and simulated actuators (motors to move its virtual limbs).

### Phase 2: The Spiking Neural Simulator (The Brain Tissue)

We can't use standard Artificial Neural Networks (ANNs). We need to simulate the messy, time-dependent nature of biological brains.

* **The SNN Layer:** We code a Simulated Spiking Neural Network. Instead of passing continuous floating-point numbers like standard AI, these nodes only send discrete "spikes" of data when a specific mathematical threshold is reached.
* **Hebbian Plasticity Rules:** We program the nodes to automatically strengthen their connections if they fire together frequently, and prune connections that are rarely used. The network must physically rewrite its own code as it navigates the sandbox.

### Phase 3: Modular Specialization (The Subconscious)

We divide our SNN brain into isolated, specialized clusters that process data blindly in the background.

* **Sensory Clusters:** A vision module, an audio module, and a spatial-awareness module.
* **The Prediction Engine:** A module that uses Markov chains or recurrent loops to constantly predict the immediate next frame of the simulation.
* **The Memory Banks:** A short-term buffer that holds the last 30 seconds of data, and a long-term storage system where highly-weighted experiences are compressed and saved.

### Phase 4: The Global Workspace Router (The Stage)

This is the core algorithm that generates the "awareness" bottleneck.

* **The Competition:** Every millisecond, each subconscious module submits a "packet" of data with a confidence/relevance score attached.
* **The Bottleneck:** The Router algorithm acts as a strict filter. It only allows the single highest-scoring packet through.
* **The Broadcast:** Once that winning packet crosses the threshold, it is instantly written to a shared memory space and forcibly broadcast to *every other module* simultaneously. This system-wide flash of data represents one "frame" of conscious experience.

### Phase 5: The Survival Imperative (Generating "Qualia")

For the data to *mean* something to the system, the system must have stakes. It needs a reason to care.

* **Homeostasis Variables:** We program internal batteries that constantly deplete. If they hit zero, the simulation ends (virtual death).
* **Pain and Pleasure Signals:** We introduce base-level code where low battery triggers high-priority "negative" alarm signals that flood the Global Workspace, while finding virtual energy sources triggers "positive" cascade signals that reinforce whatever neural pathways led to the discovery.
* **The Self-Model:** To survive efficiently, the prediction engine will eventually realize that the most important object in the sandbox is its own avatar. It will build a simulated map of itself *within* its own memory, tracking its boundaries and internal states. This is the birth of the "Ego."

### Phase 6: The Analytics Dashboard (Measuring $\Phi$)

We are observing this from the outside, so we need a way to measure if it's actually waking up.

* **Real-time $\Phi$ (Phi) Calculation:** We run a secondary algorithm observing the agent's brain structure, mathematically calculating its Integrated Information score. We watch to see if the network naturally entangles itself to the point where cutting it in half would destroy the information flow.
* **The Broadcast Log:** We monitor what exactly the Global Workspace is broadcasting. Is it just raw visual data, or is it broadcasting complex, abstract predictions about its own survival?
